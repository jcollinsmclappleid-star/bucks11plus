===========================================================
K) ANALYTICS + FORECAST ENGINE (HIGH-CONVERSION + CREDIBLE)
===========================================================

Compute on submit (server-side in /lib/forecast):

Required computed metrics:
1) score_total (# questions)
2) score_raw (# correct)
3) accuracy_total = score_raw/score_total
4) accuracy_by_section
5) accuracy_by_skill
6) avg_time_per_q overall + by section + by skill
7) completion_ratio (answered/total)
8) easy_miss_count (# incorrect where difficulty<=2)
9) volatility_index:
   - stddev(time_spent_seconds)
   - correctness streak volatility (optional)
10) pace_flags per section/skill (slow if avg_time > expected)

FORECAST MODEL V1 (implement exactly; keep transparent):

Constants:
- BENCHMARK = 121
- expected_time_per_q:
  VR=35s, NVR=35s, Maths=45s (adjustable constants)

Compute:
accuracy_pct = accuracy_total*100

penalty = 0

If completion_ratio < 1:
  penalty += (1 - completion_ratio) * 12   (up to 12 points)

If avg_time_per_q > expected_time_overall:
  over = (avg_time_per_q - expected_time_overall)
  penalty += min(6, over/10)  (max 6 points)

easy_penalty = min(6, easy_miss_count * 1.2)

mid_score = round( 100 + (accuracy_pct - 50) * 0.7 - penalty - easy_penalty )

Clamp mid_score 80..140.

Range width:
range_width = 4
If volatility_index high: range_width += 2
If only one full diagnostic/mock completed historically: range_width += 2
If completion_ratio < 1: range_width += 2

low = mid_score - range_width
high = mid_score + range_width

Band:
- On Track if low >= 121
- Borderline — Within Reach if low < 121 <= high
- Clear Improvement Opportunity if high < 121

Gap statement:
- If high < 121: “Estimated gap to 121: {121-high}–{121-mid_score} marks”
- If intersects: “Within 1–3 marks of 121”
- If low > 121: “Above benchmark — maintain and improve consistency”

Confidence:
- Low if only mini diagnostic or 1 attempt total
- Medium if 1 full diagnostic/mock
- High if 2+ full diagnostics/mocks and low volatility

Likelihood band (paid diagnostics only):
- High likelihood if mid_score >= 125 AND confidence >= Medium
- Moderate likelihood if 118–124
- Low likelihood otherwise

Impact scoring:
For each skill:
impact = (1 - accuracy_skill) * 100
+ (pace_flag ? 15 : 0)
+ (easy_miss_skill ? 10 : 0)
Sort desc, take top 3.

Improvement simulation (conversion/hope):
For top impact skill:
simulate +10% accuracy on that skill (cap 100).
Adjust accuracy_total slightly and recompute mid_score delta approximately.
Display:
“If {skill} accuracy improves by ~10%, projected range shifts to X–Y.”

Store in attempts.forecast JSON:
{
 benchmark: 121,
 projected_low, projected_high, mid_score,
 readiness_band, confidence,
 likelihood_band (null for free mini),
 gap_statement,
 top_focus_skills: [{skill_id, name, impact, accuracy, pace_flag}],
 simulation: {skill_id, uplift_pct:10, new_low, new_high}
}
